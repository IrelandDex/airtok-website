故障排查

两台设备互相看不到

AirTok 通过局域网自动发现设备，请先确认：

  • 两台设备连的是同一个 Wi-Fi（不是同一运营商不同热点）
  • 路由器没有开启"AP 隔离" / "客户端隔离"（部分酒店 / 公司 Wi-Fi 会开，会屏蔽设备互相发现）
  • iOS 首次启动时弹出的"本地网络"权限要允许，否则 iOS 会静默屏蔽 AirTok 的发现包
  • 防火墙没有拦截 53317（TCP）和 53318（UDP）端口

以上都检查过仍看不到，可以用"扫二维码连接"的方式手动配对。

iOS 看不到 Mac / Windows 设备

iOS 14 起对多播权限管得很严，AirTok 的多播权限申请正在 Apple 审核中。在权限到位前：

  • 首次打开 iOS 时允许"本地网络"权限
  • Mac / Windows 这一端能正常看到 iOS（单向工作）
  • 从 Mac / Windows 发起传输，iOS 会收到弹窗
  • 或者用扫二维码的方式手动配对
传输到一半中断了怎么办

AirTok 支持断点续传，不用担心丢文件：

  • 中断的文件会在接收端以 .part 后缀临时保存
  • 未完成的任务会显示在"设备"页的"未完成"区域
  • 点击"继续"即可从中断位置恢复
  • 不想继续可以点"取消"，AirTok 会清理两端的 .part 残留
扫了二维码但连不上
  • 确认二维码是从同一个局域网内的另一台 AirTok 生成的
  • 二维码有时效（10 分钟），过期后需要重新生成
  • 如果两台设备之前信任过但现在装过新版本，对方的证书指纹可能变了，需要先在"设置 → 已信任设备"里移除旧记录再重新配对
传输速度明显变慢
  • Wi-Fi 信号弱 / 拥堵是最常见原因，可以把两台设备都靠近路由器
  • 某些 5GHz 路由器对 UDP 多播有限速，改用有线或靠近 AP 会快
  • 同时开了 SHA-256 完整性校验会有一点性能开销（约 5-10%），追求极速时可以在设置里关闭
  • 大量小文件本身比大文件慢，因为每个文件有独立的握手开销

进阶用法

如何跨平台传 Live Photo / 动态照片

AirTok 会自动识别并保留动态效果，无需额外操作：

  • iOS → Android：自动转成目标手机原生支持的动态照片格式（华为 / 三星 / Google / 小米 / OPPO / vivo 等）
  • Android → iOS：自动重组为 Apple 认可的 Live Photo 对
  • 保存到相册后按住照片即可播放动效
如何移除已信任的设备

进入"设置 → 已信任设备"，点击要移除的设备条目。移除后对方再发文件会被要求重新确认。

这个功能在以下情况很有用：

  • 设备易主（二手出售前清理）
  • 对方重装了 AirTok，证书指纹变了
  • 临时信任了一台共享电脑，事后取消
如何查看 AirTok 版本号

打开应用后点右下角"关于"页，顶部显示版本号。发 bug 报告时请把版本号一起附上。

卸载

如何在各平台卸载
  • macOS：从"应用程序"拖到废纸篓。应用数据在 ~/Library/Application Support/AirTok/（默认保留），接收的文件在"下载"目录。
  • Windows：从设置 → 应用 → 已安装应用找到 AirTok 卸载。应用数据在 %APPDATA%\AirTok\。
  • Linux：deb 装的用 apt remove airtok；AppImage 直接删掉文件即可。应用数据在 ~/.local/share/AirTok/。
  • iOS：长按图标 → 删除 App。iOS 会同时清除所有应用数据。
  • Android：长按图标 → 卸载。部分手机会询问是否同时删除应用数据。
Windows / Linux 需要装什么依赖

AirTok 把所有依赖都打包进了安装包：

  • Windows：zip 解压直接运行，或安装 exe
  • Linux：.deb 或 AppImage，装好即用，需要系统有 GTK 3（大部分发行版自带）
