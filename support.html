常见问题

两台设备互相看不到怎么办？

AirTok 通过局域网自动发现设备，请先确认：

  • 两台设备连的是同一个 Wi-Fi（不是同一运营商不同热点）
  • 路由器没有开启 "AP 隔离" / "客户端隔离"（部分酒店/公司 Wi-Fi 会开，会屏蔽设备互相发现）
  • iOS 首次启动时弹出的"本地网络"权限要允许，否则 iOS 会静默屏蔽 AirTok 的发现包
  • 防火墙没有拦截 53317（TCP）和 53318（UDP）端口

以上都检查过仍看不到，可以用"扫二维码连接"的手动方式直接配对。

iOS 看不到 Mac / Windows 设备？

iOS 14 以后对多播权限管得很严，AirTok 的多播权限申请正在审核中。在多播权限到位前：

  • 首次打开 iOS 允许"本地网络"权限
  • Mac / Windows 这一端能正常看到 iOS（单向工作）
  • 点 Mac / Windows 端的设备发起传输，iOS 会收到弹窗
  • 或者用扫二维码的方式手动配对
传输大文件到一半中断了，会丢吗？

不会丢。AirTok 支持断点续传：

  • 中断的文件会在接收端以 .part 后缀临时保存
  • 下次再发同一个文件，AirTok 会自动从中断位置继续传
  • 未完成的任务会显示在"设备"页的"未完成"区域，可以主动点"继续"重试
Live Photo 跨平台传输会保留动态效果吗？

会。AirTok 是目前极少数支持 iOS Live Photo 与 Android 动态照片（华为、三星、Google、小米等）双向互转的工具。

  • iOS → Android：自动转成目标手机原生支持的动态照片格式
  • Android → iOS：自动重组为 Apple 认可的 Live Photo 对
  • 保存到相册后可以按住播放动效
传输是加密的吗？安全吗？

是。AirTok 采用多层安全措施：

  • 所有传输走 TLS 1.3 端到端加密
  • 设备间互相验证证书指纹
  • 首次连接采用 Trust-On-First-Use（首次信任），之后指纹变化会警告
  • 可以在"设置"里开启 SHA-256 文件完整性校验
  • 可以在"设置"里开启"接收前确认"防止陌生设备推送文件
传输的文件会经过服务器吗？

不会。AirTok 纯点对点 LAN 传输，文件直接从发送设备发到接收设备，不经过任何云或中转服务器。离线 / 断网环境只要两台设备连同一路由器就能传。详见隐私政策。

忘记信任过的设备怎么办？

在"设置 → 已信任设备"里可以看到所有已信任的设备列表，点击即可移除。移除后对方再发文件会被要求重新确认。

如何在 Mac 上卸载 AirTok？
  • 从"应用程序"文件夹里直接拖到废纸篓即可
  • 应用数据存在 ~/Library/Application Support/AirTok/，默认会保留（便于重装后恢复信任设备），可以手动删除
  • 已接收的文件在你的"下载"目录下，卸载不会删除
为什么 Windows / Linux 版要额外装什么东西？

不用装额外的。AirTok 把所有依赖都打包进了安装包：

  • Windows：zip 解压直接运行，或安装 exe
  • Linux：.deb 或 AppImage，装好即用，需要系统有 GTK 3（大部分发行版自带）
