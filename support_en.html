Troubleshooting

Two devices can't see each other

AirTok discovers devices over the local network. Please verify:

  • Both devices are on the same Wi-Fi network (not just the same provider)
  • The router does not have AP isolation / client isolation enabled (common on hotel / corporate Wi-Fi; blocks device-to-device visibility)
  • On iOS, you allowed the "Local Network" permission prompt at first launch — iOS silently drops discovery packets otherwise
  • Your firewall is not blocking ports 53317 (TCP) and 53318 (UDP)

If nothing works, use the QR-code connect option to pair manually.

iOS can't see Mac / Windows devices

Since iOS 14, multicast access requires a special Apple-granted entitlement. AirTok's multicast entitlement request is pending review. In the meantime:

  • Grant the "Local Network" permission on first launch
  • Your Mac / Windows can still see the iOS device (one-way visibility)
  • Start the transfer from Mac / Windows; iOS will receive a prompt
  • Or use QR code pairing
A transfer was interrupted

AirTok supports resumable transfers — you won't lose the file:

  • Partially received files are stored with a .part suffix
  • Unfinished tasks appear in the "Unfinished" section on the Devices page
  • Tap "Continue" to resume from where it stopped
  • Tap "Cancel" to drop the task — AirTok cleans up .part remnants on both sides
QR code scanned but devices won't connect
  • Confirm the QR code was generated from another AirTok device on the same local network
  • QR codes expire after 10 minutes — regenerate if needed
  • If the two devices were previously trusted but one of them has been reinstalled, the cert fingerprint has changed. Remove the old trusted-device entry in Settings and pair again
Transfer speed is noticeably slow
  • Weak / congested Wi-Fi is the most common cause; move both devices closer to the router
  • Some 5 GHz routers rate-limit UDP multicast — wired or near-AP gives best speed
  • SHA-256 integrity check adds ~5–10% overhead; you can disable it in Settings for pure speed
  • Many small files are inherently slower than one large file due to per-file handshake overhead

Advanced

How do Live Photos / Motion Photos transfer cross-platform

AirTok auto-detects and preserves motion, nothing manual needed:

  • iOS → Android: auto-converts to the target phone's native format (Huawei / Samsung / Google / Xiaomi / OPPO / vivo, etc.)
  • Android → iOS: reconstructs a proper Apple Live Photo pair
  • Playable with long-press after saving to the camera roll
How to remove a trusted device

Go to Settings → Trusted Devices and tap the entry. Next send from that device will require re-confirmation.

Useful when:

  • The device changes hands (cleaning before selling)
  • The other side reinstalled AirTok and the cert fingerprint changed
  • You temporarily trusted a shared computer and want to revoke access
How to find my AirTok version

Open the app and tap the About tab (bottom right). Version number is at the top. Please include it in any bug report.

Uninstall

How to uninstall on each platform
  • macOS: Drag from Applications to the Trash. App data lives at ~/Library/Application Support/AirTok/ (kept by default); received files are in Downloads.
  • Windows: Settings → Apps → Installed apps → AirTok → Uninstall. App data lives at %APPDATA%\AirTok\.
  • Linux: apt remove airtok for the .deb, or just delete the AppImage file. Data lives at ~/.local/share/AirTok/.
  • iOS: Long-press the icon → Delete App. iOS removes all app data at the same time.
  • Android: Long-press the icon → Uninstall. Some OEMs ask whether to clear app data too.
Do Windows / Linux builds need extra dependencies

No. AirTok bundles everything it needs:

  • Windows: unzip and run, or install via the .exe
  • Linux: .deb or AppImage, ready out of the box; needs GTK 3 (shipped with most distros)
