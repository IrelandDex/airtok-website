隐私政策

生效日期：2026 年 4 月 17 日 · 最后更新：2026 年 5 月 5 日

简要说明：文件内容始终在您的设备之间直接流动——我们看不到这些内容，也不会在我们的服务器上存储。我们不集成任何用于数据分析或追踪的第三方 SDK。我们服务器上唯一收集的数据，是支撑可选的账号功能（登录、付费权益、跨网络传输）所必需的最少数据。

1. 应用简介

AirTok（以下简称"本应用"）是一款跨平台端到端文件传输工具，支持 iOS、macOS、Android、Windows 和 Linux。本应用由独立开发者 Dex Li（以下简称"我们"）开发和运营。

本应用提供两种传输模式：

  • 局域网（LAN）模式：当两台设备处于同一 Wi-Fi 时，文件在设备之间直接传输，不经任何服务器，不消耗流量。
  • 跨网络（WAN）模式：当设备处于不同网络时，本应用在您的两台设备之间建立端到端加密通道。文件内容仍然只有您的两台设备能解密，我们和任何中间服务器都无法读取。

我们设计 AirTok 时遵循"最小化处理用户数据"的原则。本隐私政策旨在透明地说明我们如何（以及如何不）处理您的数据。

2. 我们不收集的数据

本应用不收集、不存储、不传输以下任何数据到我们的服务器或任何第三方：

  • 您传输的文件内容或文件名
  • 广告 / 追踪标识符（IDFA、AAID 等）
  • 精确或粗略位置信息
  • 使用行为数据或分析统计
  • 通讯录、日历、浏览记录等个人敏感数据
  • 崩溃日志（不会自动上报）

3. 文件如何传输

局域网模式（同一 Wi-Fi）

  • 文件通过 TCP 点对点连接在设备间直接传输，不经过任何云服务器
  • TLS 1.3 加密保护数据在传输过程中的安全
  • 传输完成后，文件保存在接收方设备的本地存储中

跨网络模式（不同网络）

  • 本应用使用我们的信令服务，帮助您的两台设备在不同网络之间互相发现。信令消息仅包含公开的连接元数据（如加密握手参数），不包含文件内容
  • 当因 NAT/防火墙限制无法建立点对点直连时，文件流量会通过 TURN 服务器中转。TURN 服务器只转发已加密的字节，无法读取文件内容
  • 端到端加密保护文件内容——只有您的两台设备持有密钥，中转服务器没有
  • 文件在信令服务器或 TURN 服务器上的任何阶段都不会被存储

4. 设备权限使用说明

本应用可能请求以下设备权限，每项权限均仅用于其声明的目的：

权限用途何时请求
本地网络在局域网内发现其他设备并传输文件首次启动时
照片库 / 媒体选择照片/视频发送，或将接收的媒体保存到相册选择发送照片或保存时
相机扫描二维码以快速连接其他设备使用扫码功能时
文件访问选择文件发送，或保存接收的文件选择发送文件时
通知展示传输进度和完成提醒首次发起传输时（Android 13+ 在首次启动时）

您可以随时在系统设置中撤销以上任何权限。撤销权限可能导致相关功能不可用，但不会影响应用的其他功能。

5. 网络通信

本地网络（局域网）

  • UDP 广播/多播（端口 53318）：用于自动发现同一网络内的其他 AirTok 设备。广播内容仅包含设备名称和连接端口，不含任何个人信息。
  • TCP 连接（端口 53317）：用于设备间的文件传输。支持 TLS 1.3 加密。
  • 本地 HTTP（默认端口 53319，若被占用则由系统分配备用端口；仅桌面端）：应用内部组件间的通信，仅监听 127.0.0.1，外部设备无法访问。

互联网端点

  • signaling.airtokapp.com（Cloudflare Workers / Durable Objects）：WebSocket 信令服务，帮助您的设备在不同网络之间建立连接。仅看到加密的握手元数据，不会看到文件内容。
  • Cloudflare TURN 服务器：仅在因网络限制无法建立点对点直连时作为中转使用。仅转发已加密的字节，无法解密。
  • api.airtokapp.com：用于账号登录、付费权益校验和版本检查的后端。仅在您主动使用账号 / 付费功能时（或本应用检查新版本时）访问。
  • Apple / Google 身份验证端点：当您选择以 Apple 或 Google 登录时，由系统直接发起请求。我们不会接触您的密码。

6. 本地数据存储

本应用在您的设备本地存储以下数据：

  • 应用设置：下载目录、设备名称、主题偏好等配置项
  • 设备信息：已发现和已信任的设备列表（设备名、IP 地址、信任状态）
  • 传输历史：近期传输记录（文件名、传输状态、时间）
  • 接收的文件：保存在您指定的下载目录中
  • TLS 证书：用于加密传输的本地密钥对

以上数据全部存储在您的设备本地，不会上传到任何服务器。卸载应用将删除除已接收文件外的所有数据。

7. 账号与登录

使用局域网传输无需登录。您可以在多台同一 Wi-Fi 下的设备上安装并使用本应用，全程无需登录。

登录账号可解锁跨网络传输和付费功能。可选的登录方式包括：

  • Sign in with Apple（iOS / macOS，使用 Apple 原生 AuthenticationServices 框架）。我们会收到您的 Apple 稳定用户 ID 以及您选择共享的邮箱地址（真实邮箱或 Apple 中继邮箱）。我们看不到您的 Apple ID 密码。
  • Sign in with Google（Android / iOS）。我们会收到您的 Google 账号邮箱地址和稳定用户 ID。我们看不到您的 Google 密码。
  • 邮箱 + 一次性验证码：您输入邮箱地址，我们向其发送一次性验证码。我们仅存储该邮箱地址。

登录后，我们的服务器存储：您的邮箱地址、稳定账号 ID、所选登录方式、账号创建与最近一次登录的时间戳。这些数据仅用于支撑账号相关的功能，并将您的多台设备关联到同一账号。

8. 第三方服务

本应用不集成任何用于数据分析、广告、追踪或崩溃报告的第三方 SDK，包括但不限于：

  • 分析/统计 SDK（如 Google Analytics、Firebase Analytics）
  • 广告 SDK（如 AdMob、Facebook Ads）
  • 归因 / 追踪 SDK（如 Adjust、AppsFlyer）
  • 崩溃报告 SDK（如 Crashlytics、Sentry）

本应用使用以下第三方服务，因为它们是核心功能不可或缺的部分：

  • Cloudflare——为跨网络传输提供信令服务和 TURN 中转（详见 §3 和 §5）。Cloudflare 处理加密握手数据，以及（中转时）端到端加密的文件字节。Cloudflare 在合同上是数据处理者，无法解密您的文件。
  • Apple Sign In（Sign in with Apple）——仅在您选择以 Apple 登录时使用。整个流程在您的设备、Apple 与我们的后端之间直接完成；我们仅接收 Apple 签发的身份令牌。
  • Google Sign In——仅在您选择以 Google 登录时使用，处理方式同上。
  • Apple StoreKit（仅 iOS）——处理付费订阅与一次性购买。付款、续订、退款完全由 Apple 账户管理，我们不经手您的支付信息（信用卡号、Apple ID 密码等永远不会触达我们的服务器）。

为校验付费状态，App 会将 Apple 返回的 交易标识符（transactionId / originalTransactionId）发送到我们的服务器。我们的服务器使用 Apple 官方 App Store Server API 验证该交易，并存储如下信息：

  • 订阅产品 ID、订阅周期开始 / 结束时间、自动续费开关状态
  • 一次性流量包（Topup）的发放时间、有效期、剩余字节数
  • 用于将上述记录关联到您账户的 Apple originalTransactionId

此外，我们会接收 Apple 通过 App Store Server Notifications V2 发送的订阅生命周期事件（成功续费 / 退款 / 取消等），以同步更新您的付费状态。

账号与权益数据仅用于账号运行、付费功能权限校验和订阅生命周期管理，不用于广告投放、用户画像分析或转售给任何第三方。仅当您主动注销账号（详见 §11）时，相关记录会进入软删除流程。

9. 数据安全

我们采取以下措施保护您的数据安全：

  • 文件传输支持 TLS 1.3 端到端加密
  • 设备间通过 TLS 证书指纹互相验证身份
  • 首次连接后锁定对方的证书指纹（Trust-On-First-Use），防止中间人攻击
  • 可选的文件完整性校验（SHA-256），确保传输内容未被篡改
  • 可选的接收确认机制，防止未经授权的设备推送文件

10. 儿童隐私

本应用不针对 13 岁以下的儿童，也不故意收集任何儿童的个人信息。如果您是家长或监护人，发现您的孩子在使用本应用时遇到隐私问题，请联系我们。

11. 您的权利与数据保留

我们在服务器上保留的数据（仅当您登录账号后）

  • 账号邮箱 + ID + 登录方式：保留至您注销账号
  • 订阅 / 付费权益记录：保留至您注销账号；注销后会保留计费争议窗口期（通常 180 天）后最终删除
  • 文件传输内容与元数据：在任何阶段都不会存储在我们的服务器上

您可以做的事

  • 注销账号：在 App 内通过 账号 → 注销账号（iOS 与 Android 均支持），或发邮件给我们。注销账号会清除您的登录身份并触发权益软删除流程。
  • 导出您的数据：发邮件联系我们，我们会提供与您账号关联的个人数据副本。
  • 退出登录：从当前设备移除会话令牌，但不会删除账号。
  • 清理本地数据：在应用设置中清除传输历史、删除已接收的文件、取消设备信任，或卸载应用以删除所有本地数据。

12. 适用法律

本隐私政策适用于以下隐私法规的要求：

  • 《中华人民共和国个人信息保护法》（PIPL）
  • 《通用数据保护条例》（GDPR，适用于欧盟用户）
  • 《加州消费者隐私法案》（CCPA，适用于加州用户）

对于使用了账号功能的用户，我们按照上述法规支持您的数据主体权利（访问、删除、导出），具体方式见 §11。

建议您定期查看本页面以了解最新的隐私保护信息。
